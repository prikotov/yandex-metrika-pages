# Yandex Metrika Pages

> Статистика по страницам сайта из Яндекс.Метрики

## Зачем это нужно

Этот skill анализирует страницы сайта. Данные помогают:

- **Найти популярные страницы** — какие страницы чаще всего просматривают
- **Определить точки входа** — с каких страниц начинается знакомство с сайтом
- **Найти проблемные места** — страницы выхода с высокими отказами

## Что вы получите

Отчёт в форматах CSV и Markdown:

| Файл | Содержание |
|------|------------|
| `pages_*.*` | Страницы с метриками: просмотры, посетители, отказы и др. |

## Зависимости

Требует установленный [yandex-metrika-core](https://github.com/prikotov/yandex-metrika-core)

## Установка

Skill совместим с различными AI-агентами. Примеры ниже даны для OpenCode — для других инструментов смотрите их документацию по установке skills.

```bash
# Сначала установите core
git clone https://github.com/prikotov/yandex-metrika-core.git .opencode/skills/yandex-metrika-core

# Затем этот skill
git clone https://github.com/prikotov/yandex-metrika-pages.git .opencode/skills/yandex-metrika-pages
```

## Использование

### Напрямую через PHP

```bash
# Топ страниц по просмотрам
php .opencode/skills/yandex-metrika-pages/pages.php

# Топ-20 страниц входа
php .opencode/skills/yandex-metrika-pages/pages.php -t entry -l 20

# Страницы выхода с наибольшим процентом отказов
php .opencode/skills/yandex-metrika-pages/pages.php --type exit --sort bounce_rate -l 15
```

### Типы отчётов (`--type` / `-t`)

| Параметр | Описание |
|----------|----------|
| `page` | Все страницы (по умолчанию) — только просмотры и посетители |
| `entry` | Страницы входа — полные метрики сессий |
| `exit` | Страницы выхода — полные метрики сессий |

### Через агента

После установки skill агент автоматически узнаёт о нём. Примеры запросов:

```
Покажи топ страниц по просмотрам
```

```
Сделай топ-10 страниц входа с наибольшим процентом отказов
```

```
Проанализируй с каких страниц люди уходят с сайта
```

## Результаты

Отчёты сохраняются в папку с датой:

```
yandex_metrika_reports/
└── 2026-03-03/
    ├── pages_2026-03-03_10-30-15.csv
    └── pages_2026-03-03_10-30-15.md
```

CSV открывается в Excel/LibreOffice, Markdown — в любом текстовом редакторе или напрямую в Obsidian.
